## Vision

### 10x Check
"AI 트레이딩 에이전트의 앱스토어" — 사이트에 접속하면 실시간 에이전트 성과가 보이고, 전략을 고르고, 지갑 연결하고, 클릭 한 번으로 Akash에 배포. 에이전트와 대화하며 실시간으로 제어.

현재: Docker CLI → 수동 설정 → 개발자만 사용 가능
목표: 웹사이트 → 지갑 연결 → 원클릭 배포 → 누구나 사용 가능

### Platonic Ideal
사용자 감정의 여정:
- 첫 방문: "와, 이게 진짜 돌아가고 있구나" (실시간 성과가 보이는 랜딩)
- 지갑 연결: "내 지갑이 바로 인식되네" (체인 자동 감지, 잔고 즉시 표시)
- 설정: "이렇게 간단해도 되나?" (슬라이더 하나로 리스크, 전략은 카드 선택)
- 배포: "진짜 한 번에 되네" (프로그레스 바가 아름답게 흘러가고, 각 단계가 뭔지 알려줌)
- 운영: "내 에이전트가 일하는 게 보인다" (트레이딩 로그가 실시간으로 스트리밍)
- 종료: "깔끔하다" (원클릭 종료, 잔여 자금 자동 회수)

### Wallet Architecture (Dual-Wallet UX)

**EVM Wallet (wagmi v2 + RainbowKit):**
- 사용자의 기존 MetaMask/WalletConnect 지갑
- Arbitrum 네트워크 중심 (USDC 잔고 확인, 에이전트 자금 전송)
- 지갑 서명으로 사용자 신원 확인 (SIWE — Sign In With Ethereum)

**Akash 배포 (Managed Wallet API — AEP-63):**
- **EUREKA:** Akash Console의 Managed Wallet REST API 사용
- AKT 불필요, Keplr 불필요, 크레딧 카드 기반 managed wallet
- `/api/deploy` route가 Akash Console API를 호출하여 배포 전체 라이프사이클 관리
- 사용자에게 USDC로 배포 비용 청구 → 플랫폼의 Akash Console 계정으로 크레딧 카드 결제
- **이점:** AKT float 관리 불필요, AKT 가격 리스크 제로, Cosmos 생태계 의존 제로
- API 엔드포인트: https://console-api.akash.network (create/list/close deployments, check balance)

**Auth (SIWE + iron-session):**
- SIWE (Sign In With Ethereum)로 지갑 서명 인증
- iron-session으로 encrypted cookie 기반 세션 관리 (24시간 TTL)
- `/api/deploy`, `/api/agent` 등 보호 route는 세션 미들웨어로 guard
- 세션에 `userAddress`, `chainId`, `sessionCreatedAt` 저장

**Hot Wallet Funding Flow:**
- 배포 시 서버가 WAIaaS로 새 hot wallet 생성, `hot_address`를 `deployments` 테이블에 저장
- 사용자가 EVM 지갑에서 USDC를 hot wallet 주소로 직접 전송 (UI에서 트랜잭션 서명)
- 에이전트가 hot wallet의 USDC를 사용하여 트레이딩
- 종료 시 hot wallet → 사용자 지갑으로 잔액 반환 (에이전트가 서명)

### Agent ↔ Dashboard Communication

Agent Container는 Neon/Redis에 직접 의존하지 않습니다. 자체 완결적으로 실행됩니다.

**데이터 흐름:**
```
Browser ←→ /api/agent (polling, 5s) ←→ Neon DB ←── /api/cron (집계)
                                                        ↑
Browser ←→ /api/deploy ←→ Akash Console API ←→ Agent Container
                              (Logs + Status API)
```

- **Agent → Dashboard (상태/로그):** Next.js `/api/agent` route가 Akash Console API의 로그/상태 엔드포인트를 통해 에이전트 정보를 가져옴. 대시보드 클라이언트가 5초 간격 polling.
- **Dashboard → Agent (채팅/명령):** 사용자 메시지를 Neon `agent_messages` 테이블에 저장. `/api/deploy`가 Akash Console API를 통해 에이전트에 전달 (컨테이너 exec 또는 환경 변수 업데이트). **동시성 보호:** atomic UPDATE RETURNING.
- **Agent Container는 외부 DB 의존성 없이** 기존 OpenClaw + a2ex + WAIaaS 스택 그대로 실행.
- **거래 데이터 집계:** Akash 로그에서 거래 기록을 파싱하여 Neon `trades` 테이블에 cron으로 적재.

### Kill Switch Sequence

```
UI "종료" 클릭
  → /api/deploy/terminate (POST)
    → 1. Neon에 shutdown 명령 INSERT
    → 2. 에이전트가 명령 수신 (polling)
    → 3. 열린 포지션 정리 (시장가 청산)
    → 4. 남은 USDC를 사용자 EVM 지갑으로 전송 (hot wallet → user address)
    → 5. 에이전트가 "shutdown_complete" 상태 POST to Neon
    → 6. 서버가 Akash close-deployment TX 실행 (AKT 에스크로 반환)
    → 7. 대시보드에 "Terminated — 잔여 자금 반환 완료" 표시
```

타임아웃: 5분 내 에이전트 응답 없으면 강제 Akash deprovision + 사용자에게 "수동 자금 회수 필요" 알림.

### Akash Deploy Error Handling

- **Bid 타임아웃:** 60초 내 bid 미수신 시 `status='failed'`, UI에 "No providers available — retry?" 표시
- **Provider dropout:** lease 생성 후 provider 오프라인 시 에러 감지, 대시보드에 표시, "Re-deploy" 버튼 제공
- **Container crash:** 에이전트가 30초 이상 heartbeat 미전송 시 `status='unhealthy'` 표시, 자동 재시작 또는 수동 조치 안내

### Neon Database Schema (v1)

```sql
-- 사용자 배포
CREATE TABLE deployments (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_address  TEXT NOT NULL,          -- EVM wallet address (SIWE 검증됨)
  akash_dseq    TEXT,                   -- Akash deployment sequence
  strategy_id   TEXT NOT NULL,          -- 선택된 전략
  config        JSONB NOT NULL,         -- 자금 한도, 리스크 파라미터
  status        TEXT NOT NULL DEFAULT 'pending',  -- pending/sdl_generated/bid_received/provider_selected/active/terminating/terminated/failed
  hot_address   TEXT,                   -- 에이전트의 hot wallet 주소
  created_at    TIMESTAMPTZ DEFAULT now(),
  terminated_at TIMESTAMPTZ
);

-- 거래 기록 (에이전트가 HTTP POST)
CREATE TABLE trades (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  deployment_id UUID REFERENCES deployments(id),
  venue         TEXT NOT NULL,          -- polymarket/hyperliquid
  action        TEXT NOT NULL,          -- buy/sell
  amount_usd    NUMERIC(12,2),
  pnl_usd       NUMERIC(12,2),
  ts            TIMESTAMPTZ DEFAULT now()
);

-- 플랫폼 통계 스냅샷 (Vercel Cron: /api/cron/stats, CRON_SECRET 인증, 5분 간격)
-- SQL: INSERT INTO stats_snapshots SELECT count(*), sum(balance), sum(volume), sum(pnl) FROM deployments JOIN trades ...
-- 동일 cron에서 strategies.performance도 갱신 (최근 7일 trades 집계)
CREATE TABLE stats_snapshots (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  total_agents  INT,
  total_aum_usd NUMERIC(14,2),
  total_volume  NUMERIC(14,2),
  total_pnl_usd NUMERIC(14,2),
  snapshot_at   TIMESTAMPTZ DEFAULT now()
);

-- 전략 목록
CREATE TABLE strategies (
  id            TEXT PRIMARY KEY,       -- e.g. "polymarket-sports-basic"
  name          TEXT NOT NULL,
  description   TEXT,
  venues        TEXT[] NOT NULL,        -- ["polymarket", "hyperliquid"]
  min_fund_usd  NUMERIC(8,2) DEFAULT 10,
  risk_level    TEXT DEFAULT 'medium',  -- low/medium/high
  config_schema JSONB,                  -- v1: 참조용. UI는 전략별 하드코딩 폼. v2: JSON Schema → 동적 폼 렌더링
  performance   JSONB,                  -- { "7d": 0.12, "30d": 0.05, sparkline: [...] }
  active        BOOLEAN DEFAULT true
);

-- 채팅/명령 (에이전트 ↔ 대시보드)
CREATE TABLE agent_messages (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  deployment_id UUID REFERENCES deployments(id),
  direction     TEXT NOT NULL,          -- 'user_to_agent' / 'agent_to_user'
  content       TEXT NOT NULL,
  processed     BOOLEAN DEFAULT false,
  ts            TIMESTAMPTZ DEFAULT now()
);
```

### Infra & Deployment

- **Frontend:** 기존 executor/landing Next.js 프로젝트 확장 (동일 Vercel 프로젝트)
- **Database:** Neon Serverless Postgres (새 프로젝트)
  - Env: `NEON_DATABASE_URL` (Vercel env vars)
  - Connection: `@neondatabase/serverless` (HTTP over WebSocket, edge-compatible)
- **Akash:** Akash Console Managed Wallet API (AEP-63) 사용
  - API key: Vercel env var `AKASH_CONSOLE_API_KEY`
  - SDL 템플릿은 `landing/lib/akash/sdl-templates/` 디렉토리에 저장
  - AKT/Keplr 불필요 — 크레딧 카드 기반 managed wallet
- **환경 변수:**
  - `NEON_DATABASE_URL` — Neon 연결 문자열
  - `UPSTASH_REDIS_URL` — Upstash Redis 연결 문자열
  - `AKASH_CONSOLE_API_KEY` — Akash Console Managed Wallet API 키
  - `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` — WalletConnect 프로젝트 ID
